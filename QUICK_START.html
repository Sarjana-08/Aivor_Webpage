🚀 Aivor Quick Start Guide

Get up and running with Aivor in minutes

Welcome to Aivor

Aivor is a fully-featured Employee Leave Management System built with vanilla HTML, CSS, and JavaScript. This guide will help you set up and customize Aivor for your organization.

📋 Initial Setup

  1. Extract Files

    Extract all Aivor files to a folder on your web server or local directory.

  2. Open in Browser

    Open index.html in any modern web browser.

  3. Login with Demo Credentials

    Use demo@aivor.com / demo123 to access the system.

  4. Test All Features

    Explore the dashboard, leave management, calendar, and attendance modules.

🎨 Customization Guide

1. Change Company Name and Branding

Edit the following in config.js:

company: { name: 'Your Company Name', tagline: 'Your Company Tagline', website: 'https://yourcompany.com', email: 'support@yourcompany.com', phone: '+1-XXX-XXX-XXXX' }

2. Update Company Colors

Modify the color scheme in styles.css:

:root { --primary-color: #YOUR_PRIMARY_COLOR; --secondary-color: #YOUR_SECONDARY_COLOR; --success-color: #YOUR_SUCCESS_COLOR; --danger-color: #YOUR_DANGER_COLOR; /* ... more colors */ }

3. Replace Logo

You have two options:

  • Option A: Replace the SVG logo in each HTML file's logo section
  • Option B: Replace SVG with an image tag:
    <img src="logo.png" alt="Company Logo">

4. Update Demo Users

Edit demo users in config.js:

demoUsers: [ { email: 'user@yourcompany.com', password: 'password123', name: 'Employee Name', userEmail: 'user@yourcompany.com', role: 'employee' } ]

5. Configure Leave Types

Customize leave types in config.js:

leaveTypes: [ { value: 'casualLeave', label: 'Casual Leave' }, { value: 'sickLeave', label: 'Sick Leave' }, // Add more... ]

6. Set Leave Balance

Configure leave balance in config.js:

leaveBalance: { totalAnnual: 20, casualLeave: 12, sickLeave: 5, paidLeave: 3 }

7. Add Company Holidays

Add holidays in config.js:

holidays: [ { date: '2026-01-01', name: 'New Year' }, { date: '2026-12-25', name: 'Christmas' }, // Add more... ]

⚙️ Configuration Quick Reference

Session Timeout

Edit in config.js under session:

inactivityTimeout: 30 * 60 * 1000, // 30 minutes

Items Per Page

ui: { itemsPerPage: 10 }

Date Format

ui: { dateFormat: 'YYYY-MM-DD' }

Enable/Disable Features

features: { leaveManagement: true, attendance: true, calendar: true, reports: true }

💡 Best Practices

  1. Always backup original files

    Keep a copy of original Aivor files before making changes.

  2. Test changes in a development environment

    Test all customizations before deploying to production.

  3. Use version control

    Track changes using Git or similar version control system.

  4. Document all customizations

    Keep track of any changes made for future reference.

  5. Clear browser cache

    Clear cache when making CSS or JavaScript changes to see updates.

⚠️ Important Security Notice

This system is a demo/prototype using browser localStorage. For production use, you MUST implement a proper backend with encryption, secure authentication, and database storage.
